Publicering

Jag spenderade äckligt lång tid med att försöka skapa Workflows, yml-filer, Secret tokens och så vidare för att publicera till Github Pages för första gången med Parcel.

Det var det absolut krångligaste och mest frustrerande jag tagit mig an sedan jag försökte få skrivrättigheter för apaches htdocs på en Mac. Så jag gav upp och valde Vercel, som jag bara har testat en gång innan och en gång är ju ingen gång.

Om Vercel

Vercel är en publiceringsplattform för statiska webbplatser och applikationer.

Språksstöd

Vercel har stöd för flera språk som bland annat JavaScript, PHP, Python, Rust och Ruby, där vissa behöver vara serverless för att fungera.

Vidare finns det också stöd för ramverk som Next.JS (deras egna skapelse), Ract, Vue, Svelte, Angular med flera.

Vercel kan också hantera runtimes som node.js och edge.

Kostnader

Vercel erbjuder tre betalalternativ: Hobby, Pro och Enterprise.

Hobby är helt gratis med obegränsade webbplatser. Med Pro får man bland annat stöd för team-kollaborationer, utökad säkerhet och bättre support, för 20 USD i månaden. Enterprise är ännu mer avancerat.

Fördelar

Det är extremt enkelt att publicera en hemsida, helt utan konfiguration. Det är så enkelt som att trycka på en knapp, vänta några sekunder och sen besöka sin webbplats.

Vercel har stöd för CI/CD automatiskt, så för varje push man gör med Git uppdateras webbplatsen.

Det är gratis att använda och har inga irriterande begränsningar även på hobbynivå.

Vercel känner av vad man har använt för package manager och tillägg, så att man inte behöver krångla.

Publicering sker på bara några sekunder.

Nackdelar

Jag tycker ärligt talat att den enda nackdelen är att Vercel bara är till statiska/serverlösa webbplatser och applikationer.

Tönk vilken lycka om man kunde lägga upp sitt API eller liknande på Vercel också.

Förutom det är jag helt nöjd med Vercel, som gör exakt vad jag vill att en molnplattform ska göra.

Tillvägagångssätt

Helst bör man, som jag, spendera minst fyra timmar med att försöka deploya till något krångligare (typ ghpages) så att man verkligen kan uppskatta hur fenomenalt smidigt det är att använda Vercel istället.

Vercel går enkelt att koppla ihop med ett GitHub-konto och känner därefter automatiskt av vilka repositories som finns att välja på.

Därmed är det löjligt enkelt att skapa ett nytt projekt i Vercel och välja vilket repo som huserar projektets filer.

Vercel känner automatiskt av vilka package managers som kan ha använts, och eventuella bundlers, och föreslår därefter bygg-kommandon (till exempel npm run build).

Det är inget som helst krångel med att ha använt parcel; Vercel känner till och med av att Parcel har använts, och visar det som ikon på översikten över publicerade sidor.

Vercel har stöd för CI/CD så för varje push man gör till sitt online-repo uppdateras den publicerade sidan.
